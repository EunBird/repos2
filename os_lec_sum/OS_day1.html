  1. 운영체제의 역할과 목적
    • 운영체제의 개념 및 역할
      • 사용자가 컴퓨터 시스템을 손쉽게 사용하도록 하며,
      • 시스템 자원을 효율적으로 관리 할 수 있도록 하는 프로그램 집함.(William Stallings)
      • 컴퓨터 하드웨어와 응용 프로그램 간의 I/F (InterFace).
      Cloud와 On-premise 는 서로 반대 개념이라 할 수 있다.
    • 운영체제의 목적
      • 편리성
        • 사용자가 프로그램을 개발하고 사용하는데에 보다 편리한 환경 제공(PC GUI 환경.). —> 생산성 기대
      • 효율성
        • 한정된 자원을 효율적으로 사용하기 위해 각 프로그램을 유기적으로 결합하여 시스템 성능을 향상시킴.
          • 처리 능력 향상 :
            • 단위 시간 내에 많은 양의 일을 처리.
            • ex) 멀티 프로그래밍, 멀티 쓰레딩
          • 응답시간 단축 :
            • 사용자가 작업을 의뢰한 후, 그 결과를 최단시간에 획득.
            • ex) 온라인 실시간, 시분할 처리 개념, 다중 처리 개념
          • 신뢰도 향상 :
            • 하드웨어, 소프트웨어의 성공적인 기능 수행.
            • ex) 시스템이 정확하게 작동되는 정도.
          • 사용가능도(가동률) 향상 :
            • 사용자가 시스템 사용 시 가장 신속하게 사용 혹은 사용 빈도가 높음.
            • ex) 부품 한개가 고장났음에도 사용이 가능하다면 사용가능도가 높은 경우이다.
            • ex) 다중 사용자 기반인 경우, 여러 사용자가 신속하게 사용가능한가 / 자원들이 쉬지 않고 사용가능한가.

  1. 운영체제의 기능과 구조
    • 운영체제의 기능
      • 입출력 장치 관리 :
        • I/O operation, 프로그램 생성 및 실행.
      • 시스템 자원 관리 :
        • 메모리와 프로세스를 제어하여 프로그램에 CPU, 램, 데이터버스 등의 시스템 자원 할당.
      • 에러 검출과 조치
        • 에러 발생에 따른 비정상 동작 조치.
        • 오류 프로세스 탐지 및 제거, 잘못 할당된 메모리 초기화 및 통지.
      • 시스템 성능 감시
      • 파일 시스템 조작
      • 네트워크 관리
      • 계정 관리
      • 보호
        1. 자원 관리 :
          1. 메모리 관리(메인 메모리, 보조기억장치), 프로세스 관리, 주변장치 관리, 파일(데이터) 관리.
        1. 시스템 관리 :
          1. 시스템 보호(사용자 권한 부여), 네트워크 관리, command interpreter
    • 운영체제의 구조
      • 5계층 : 파일 관리
        • 파일의 생성, 소멸, 파일 열고 닫기, 유지, 관리.
      • 4계층 : 주변장치 관리(입출력 관리)
        • 주변장치 상태파악, 입출력 장치 스케줄링(키보드, 마우스, 프린터).
      • 3계층 : 프로세스 관리(상위 단계)
        • 프로세스 생성, 제거, 메시지 전달, 시작, 정지.
      • 2계층 : 메모리 관리
        • 메모리(RAM, ROM)의 할당 , 회수 기능.
      • 1계층 : 프로세스 관리(하위 단계)
        • 동기화 및 프로세스 스케줄링.
      • 하드웨어

    • 커널
      • 하드웨어 특성으로부터 프로그램들을 격리시키고, 하드웨어와 직접적으로 상호작용함으로서 프로그램에 일관적인 서비스 제공.
      • 핵심 기능 :
        • 프로세스, 파일 관리
        • 입출력 장치 관리
        • 메모리 관리
        • 시스템 호출 인터페이스
      • 부팅 시 가장 먼저 실행에 필요한 다양한 서비스를 제공다양한 서비스를 제공.
      • 실행 프로세스를 스케줄링하는 역할.

    • 프로세스 관리
    • 메모리 관리
    • 주기억장치 관리
    • 보조기억장치 관리
    1. ROM : Read Only Memory
      1. 기억된 메모리를 바꿀수 없고 읽기만 가능(반영구적).
      1. masked ROM : 제조 과정에서 이미 내용을 기억시킨 메모리.
      1. PROM : 1회에 한해서 새로운 내용을 기록할 수 있는 롬.
      1. EPROM : 필요할 때 기억된 내용을 지우고 다른 내용을 기록할 수 있는 롬.
    1. RAM : Random Access Memory
      1. 읽고 쓰기가 가능한 주기억 장치.
      1. 휘발성 메모리
      1. DRAM : 정보를  축전기에 저장하는 기억 장치이지만 결국 축전기가 전자를 누전하므로 기억된 정보를 잃게 되는 메모리.
      1. SRAM : 기억 장치에 전원이 공급되는 한 그 내용이 계속 보존되는 메모리.

    • 주변 장치 관리(입출력 관리)
      • 장치 구동기(Device Driver)
        • 운영체제와 응용프로그램 및 하드웨어 간의 인터페이스 담당.
        • 장치 제어 명령어 생성 및 수행 여부 점검.
      • 데이터의 입출력 방식
        • 프로그램에 의한 입출력
          • 데이터의 입출력 동작이 CPU가 수행하는 프로그램 I/O 명령에 의해 수행.
          • CPU 가 주변장치의 상태를 계속 감시해야 함.
        • 인터럽트
          • 입출력 장치가 데이터를 전송할 준비가 되면 CPU에 인터럽트를 발생.
        • DMA(Direct Memory Access)
          • CPU를 거치지 않고, 주변장치와 메모리 사이에 직접 데이터를 전달하도록 제어하는 인터페이스 방식.
          1. 직접 입/출력(Direct I/O) :
            1. 입력 데이터 : 장치제어기 → cpu레지스터 → 주기억장치 전송.
            1. 출력 데이터 : 주기억장치 → cpu레지스터 → 장치제어기 전송.
            1. ex) Polling 방식.
          1. 간접 입/출력(Indirect I/O) :
            1. CPU가 직접 입/출력을 담당하지 않고 “인터럽트”를 사용하여 전용 입/출력 프로세서인 DMA나 Channel을 사용하는 방식.

      • 버퍼링 & 스풀링
        • 버퍼링(Buffering)
          • 입출력 장치와 보조 기억장치의 처리 속도가 CPU보다 느린 문제를 보완하는 방법 중 하나.
          • CPU가 하나의 연산을 시작하자마자, 입출력 장치가 다음에 필요한 레코드를 미리 읽어서 주기억장치에 저장함으로써 CPU의 대기시간 절감.
        • 스풀링(Spooling)
          • 버퍼링과 같은 개념이지만, 스풀링은 디스크를 매우 큰 버퍼처럼 사용.
    • 파일 관리
      • 파일에 대한 조작, 저장 방식, 접근 방법에 대한 관리 수행.

  1. 운영체제의 기술 발전 흐름
    • 단순 모니터 - 1940년대
      • 컴퓨터 오퍼레이터가 할 일 자동 수행.
    • 일괄처리 시스템(Batch Processing system) - 1950년대
      • 여러 작업을 단일 작업으로 묶어 일괄처리하는 시스템.
      • 보조기억장치가 느려 시간이 오래걸림.
      • 이를 보완하기 위해 버퍼링, 스풀링 개념이 도입.
    • 다중 프로그래밍 시스템 - 1960년대
      • 속도차이에 따른 비효율 문제 개선.
      • 주기억장치 내에 2개 이상의 프로그램을 적재하여 하나가 실행될 경우, 입출력 장치 등의 조작으로 CPU 유휴시간이 발생하면 운영체제가 다른 프로그램을 실행.
    • 시분할 처리 시스템(TSS : Time-sharing system)
      • 프로세스의 사용시간을 분할하여 다중 사용자가 CPU를 순환하여 이용.(Time slice)
    • 분산 처리 시스템(DPS : Distributed processing system)
      • 하나의 대형 컴퓨터에서 처리하던 기능을 지역적, 기능적으로 분담하여 네트워크를 통래 처리.
      • 목적 :
        1. 자원 공유
        1. 계산 속도 향상
        1. 신뢰도 향상
        1. 통신 기능
    • 다중 처리 시스템(Multiprocessing system)
      • 여러 작업을 하나의 시스템(다수의 CPU)에서 동시 처리가능한 시스템
      • 여러개의 프로세스가 메모리에 저장되어 자원이 할당될 때마다 다시 실행되면서 처리를 완료할 때까지 메모리에 존재하는 시스템.
    • 실시간 시스템(Real time system)
      • 제어 대상으로부터 처리 대상이 발생하면 즉시 처리하는 것.
      • 자료가 발생한 지점에서 단말기(pc)를 통해 직접 입/ 출력되므로 사용자의 노력이 절감된다.
      • 처리 시간 및 비용이 단축.
      • 자료가 무작위로 도착하기 때문에 입/출력 자료의 일시저장과 대기 필요.
        • 경성 : 작업의 실행 시작이나 완료에 대한 시간 제약조건을 지키지 못하는 경우 시스템에 치명적인 영향을 주는 시스템.
        • 연성 : 작업실행에 대한 시간 제약이 있지만, 이를 지키지 못하더라도 전체 시스템에 큰 영향을 끼치지 않는 시스템

  1. 운영체제의 주요 구성 기술
    • 프로세스 관리
      • 프로세스
        • 현재 수행 상태에 있는 프로그램.
        • 여러 명령어, 데이터, 카운터, CPU 레지스터, 루틴인자, 복귀주소, 변수 등 데이터 스택 포함.
      • 프로세스의 상태 변화
        • 생성(New) :
          • 새로 만들어졌지만, 운영체제에 의해 실행 가능한 프로세스 집합에 포함되지 않은 상태.
        • 준비(Ready) :
          • 프로세스가 실행되고 있지 않지만 즉시 CPU를 사용할 수 있고 사용을 기다리는 상태.
        • 실행(Running) :
          • 현재 CPU를 차지하여 동작중인 상태.
        • 대기(Block / Nating) :
          • 어떤 사건이 발생하기 전까지 실행될 수 없는 상태.
        • 보류(Hold / Terminated) :
          • 프로세스가 디스크 등에 보관되어 있는 상태.
        • 교착(Deadlock / Suspend) :
          • 결코 발생할 수 없는 사건을 기다리는 상태.
        • 종료(Exit) :
          • 운영체제에 의해 프로세스 집합에서 해제된 상태.

      • 프로세스 제어 블록(PCB)
        • 운영체제가 프로세스에 대한 중요한 정보를 저장한 저장소.
        • 운영체제는 프로세스의 실행 중단 시, 중단 후 재실행 시 등의 경우에 PCB 참조.
      • 쓰레드 & 태스크(Thread & Task)
        • 쓰레드 : 프로세스를 수행하는 데 필요한 가능한 많은 하위 작업들 중 하나.
        • 태스크 : 자원 할당의 단위로 프로세스와 유사한 개념.
        • ex) 스케줄러 → 태스크를 수행하는 가장 작은 단위가 쓰레드
        • 프로세스 : 쓰레드의 집합체.
      • 프로세스 스케줄링 → (우선순위 결정.)
        • 멀티 프로세스에서는 CPU를 사용하는 프로세스 결정이 문제.
        • 스와핑 : 메모리에서 프로세스를 제거하는 것이 효율적인 경우, 프로세스를 제거하였다가, 차후 이 프로세스를 다시 호출하여 중단되었던 지점부터 재개하는 것.
    • 스케줄링 및 알고리즘
      • 선점 스케줄링 :
        • 다른 프로세스가 선점 가능.
        • RR, SRT, MFQ
      • 비선점 스케줄링 :
        • 한 프로세스가 끝날 때까지 선점 불가.
        • SJF, FIFO, HRU

    1. FIFO :
      1. FCFS 라고도 함.
      1. First Come First Served
    1. SJF :
      1. 최소작업 우선 스케줄링
      1. 각 작업의 실행 시간을 이용하여 실행 시간이 가장 짧은 작업에 할당하는 방법이다.
    1. SRT :
      1. Shortest Remaining Time
      1. 진행 중인 프로세스가 있어도 sleep시키고 최단 잔여시간 프로세스를 먼저 할당한다.
    1. Priority
      1. Idenfinite Blocking :
        1. 무기한 봉쇄.
        1. 실행 준비는 되어있으나 CPU를 사용하지 못하는 프로세스를 CPU가 무기한 대기하는 상태.
      1. Starvation :
        1. 기아 상태.
        1. 특정 자원 목록에 대해 권한을 얻지 못해 영원히 얻지 못하여 자신의 일을 하지 못하는 상태.
    1. Round-Robin :
      1. cpu 사용시간이 랜덤한 프로세스들이 섞여 있을 때 유용.
      1. 응답시간이 빨라짐.
      1. 공정한 스케줄링.
      1. 프로세스들 사이에 우선순위를 두지 않고, 순서대로 시간단위로 CPU를 할당하는 방식.
    1. Multileval Feedback :
      1. 다단계 큐 스케줄링
      1. 커널 내의 준비 큐를 여러 개의 큐로 분리하여 큐 사이에 우선순위를 부여하는 스케줄링.
